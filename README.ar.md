## لماذا ViLog؟

إذا سبق لك أن فتحت ملف سجلات بحجم 500 ميجابايت في محرر نصوص ورأيته يتجمد، فإن ViLog صُنع من أجلك. مُصمم من الصفر لـ **تحليل السجلات على نطاق واسع**، يجمع بين عرض Canvas المُسرّع بالـ GPU، وعمال الويب متعددي المسارات، وتحسينات على مستوى الخوارزميات (Aho-Corasick، WASM) لتقديم تصفية فورية وتمرير سلس في ملفات تحتوي على ملايين الأسطر.

## الميزات

### أداء فائق السرعة

| الميزة | التفاصيل |
|--------|---------|
| **عرض Canvas** | عرض سجلات مُسرّع بالـ GPU بدلاً من عُقد DOM — يتعامل مع ملايين الأسطر بسهولة |
| **التمرير الافتراضي** | يتم عرض الأسطر المرئية فقط. مرر عبر أكثر من 10 مليون سطر بدون تأخير |
| **تصفية متعددة المسارات** | عمال ويب متوازيين يوزعون التصفية على أنوية المعالج |
| **خوارزمية Aho-Corasick** | مطابقة أنماط متعددة بتعقيد زمني O(n+z) — يصفّي أكثر من 10 كلمات مفتاحية في وقت واحد |
| **بحث WebAssembly** | أداء مطابقة سلاسل قريب من الأصلي عبر وحدات WASM |
| **فلتر ذكي هجين** | اختيار تلقائي بين ripgrep (الملفات الكبيرة) وعمال JS (الملفات الصغيرة) |
| **تخزين مؤقت لبيانات الأسطر** | التصفية المتكررة لنفس الملف تتخطى نقل البيانات — يتم إرسال الكلمات المفتاحية فقط إلى العمال |

### تصفية وبحث قويان

- **تصفية متعددة الكلمات المفتاحية** — افصل الكلمات المفتاحية بـ `|`، وتهرب من الأنبوب الحرفي بـ `\|`
- **دعم التعبيرات النمطية** — دعم كامل لـ JavaScript regex في التصفية والبحث
- **تصفية من مستويين** — فلتر أساسي + فلتر ثانوي داخل النتائج
- **سجل التصفية** — سجل كلمات مفتاحية دائم مع مطابقة ضبابية (مبني على IndexedDB)
- **تمييز الكلمات المفتاحية** — 10 ألوان مُعدة مسبقاً + منتقي ألوان مخصص
- **استبعاد الأسطر** — انقر بزر الماوس الأيمن لاستبعاد الأسطر المطابقة من النتائج
- **تنقل البحث** — Enter/Shift+Enter للتنقل بين المطابقات

### إدارة الملفات

- **شريط جانبي لشجرة الملفات** — اسحب وأفلت الملفات أو المجلدات أو الأرشيفات مباشرة
- **تصفح الأرشيفات** — ZIP، 7z، RAR، tar.gz — تصفح المحتويات بدون استخراج
- **خادم ملفات بعيد** — اتصل بالأجهزة البعيدة عبر خادم HTTP مدمج بلغة C (تجمع مسارات، تزامن عالي)
- **مشاركة محلية** — شارك المجلدات المحلية مع زملائك عبر الشبكة المحلية
- **لصق من الحافظة** — الصق الملفات مباشرة باستخدام Ctrl+V
- **عرض جدول CSV/TSV** — تحليل وعرض البيانات المهيكلة في جداول قابلة للفرز
- **تكامل Everything** — بحث فوري عن الملفات على ويندوز عبر Everything HTTP API
- **تكامل Ripgrep** — بحث نصي أسرع بـ 20-100 مرة في الملفات الكبيرة

### تصور البيانات

- **رسم مخططات CSV** — مخططات خطية تفاعلية مع تكبير وتحريك واختيار أعمدة
- **محلل Vlog** — محلل متخصص لسجلات تشخيص البطارية/الجهاز (21 حقلاً) مع تصور
- **محدد الأعمدة** — الاحتفاظ بأعمدة معينة أو إزالتها في عرض الجدول
- **تصدير** — نسخ النتائج المُصفاة أو تصديرها كـ HTML

### مساحة العمل والإنتاجية

- **نوافذ متعددة** — افتح ملفات سجلات متعددة في نوافذ منفصلة، بدّل بـ Alt+1~9
- **إشارات مرجعية** — علّم الأسطر المهمة وانتقل بينها
- **الانتقال إلى سطر** — اقفز فوراً إلى أي رقم سطر
- **روابط سريعة** — إشارات مرجعية للمواقع المتكررة (لوحة ويب مدمجة)
- **مساعد ذكاء اصطناعي** — لوحة دردشة AI مدمجة للمساعدة في تحليل السجلات
- **سجل منفذ تسلسلي UART** — نافذة مراقبة سجلات المنفذ التسلسلي
- **تحجيم الخط** — Ctrl+عجلة للتكبير، Alt+عجلة للتمرير الأفقي
- **مراقبة النظام** — عرض في الوقت الفعلي لوحدة المعالجة والذاكرة وذاكرة التطبيق
- **طرفية مدمجة** — افتح الطرفية مباشرة من التطبيق

### اختصارات لوحة المفاتيح

| الاختصار | الإجراء |
|---------|---------|
| `F` | التركيز على مربع التصفية في شريط الأدوات |
| `f` | فتح حوار التصفية |
| `Ctrl+F` | التركيز على مربع البحث |
| `Ctrl+H` | تبديل لوحة نتائج التصفية |
| `Ctrl+G` | تبديل شجرة الملفات العائمة |
| `Shift+W` | تبديل تكبير لوحة التصفية |
| `Alt+X` | تبديل ملء الشاشة |
| `Alt+1~9` | الانتقال إلى النافذة N |
| `Ctrl+Tab` | التبديل الدائري بين النوافذ |
| `Ctrl+Shift+T` | نافذة جديدة |
| `Ctrl+عجلة` | تكبير الخط |
| `Alt+عجلة` | تمرير أفقي |

### المجموعة التقنية

| المكون | التقنية |
|--------|---------|
| الإطار | Electron 28+ |
| العرض | Canvas API (تسريع GPU) |
| تعدد المسارات | Web Workers (تصفية متوازية) |
| البحث الأصلي | WebAssembly (مُترجم من C) |
| مطابقة الأنماط المتعددة | خوارزمية Aho-Corasick |
| البحث الخارجي | ripgrep، Everything SDK |
| الخادم البعيد | C + pthread (32 مسار، 4096 اتصال) |
| تحليل البيانات | PapaParse (CSV)، محلل Vlog مخصص |
| التصور | Chart.js + ملحق التكبير |
| التخزين | IndexedDB (سجل التصفية، الإشارات المرجعية) |

## البدء السريع

### المتطلبات الأساسية

- [Node.js](https://nodejs.org/) 18+
- [Electron](https://www.electronjs.org/) 28+
- (اختياري) [7-Zip](https://www.7-zip.org/) لتصفح الأرشيفات
- (اختياري) [ripgrep](https://github.com/BurntSushi/ripgrep) للبحث المُسرّع
- (اختياري) [Everything](https://www.voidtools.com/) للبحث الفوري عن الملفات على ويندوز

### التثبيت والتشغيل

<div dir="ltr">

```bash
# استنساخ المستودع
git clone https://github.com/ranxuefeng2022/ViLog.git
cd ViLog

# تثبيت التبعيات
cd jscode
npm install

# تشغيل التطبيق
npm start
```

</div>

### بناء خادم C (اختياري — لتصفح الملفات البعيدة)

<div dir="ltr">

```bash
cd server
gcc -o log_server log_server.c -lpthread -O2 -D_GNU_SOURCE

# التشغيل على المنفذ 8082
./log_server 8082 /path/to/logs
```

</div>

## معايير الأداء

| السيناريو | الأسطر | حجم الملف | وقت التصفية | FPS التمرير |
|----------|--------|----------|-------------|-------------|
| ملف واحد | 1M | 200 ميجابايت | ~0.3 ثانية | 60 |
| تصفية متعددة الكلمات (5 كلمات) | 1M | 200 ميجابايت | ~0.5 ثانية | 60 |
| دمج 10 ملفات | 5M | 1 جيجابايت | ~1.2 ثانية | 60 |
| وضع Ripgrep الهجين | 5M | 1 جيجابايت | ~0.2 ثانية | 60 |

*بيئة الاختبار: Intel i7-12700، 32 جيجابايت RAM، NVMe SSD. النتائج قد تختلف.*

## حالات الاستخدام

- **تطوير الأنظمة المضمنة/IoT** — تحليل سجلات الأجهزة، تشخيص البطارية (تنسيق vlog)
- **إدارة الخوادم** — تصفح السجلات البعيدة عبر خادم HTTP المدمج
- **ضمان الجودة/الاختبار** — مقارنة سجلات متعددة الملفات بنوافذ جنباً إلى جنب
- **تطوير الأجهزة المحمولة** — تحليل Android logcat، سجلات النواة، dmesg
- **تحليل البيانات** — تحليل CSV/TSV مع تصور تفاعلي للمخططات
