## Почему ViLog?

Если вы когда-нибудь открывали 500 МБ файл логов в текстовом редакторе и видели, как он зависает — ViLog для вас. Созданный с нуля для **масштабного анализа логов**, он объединяет рендеринг Canvas с GPU-ускорением, многопоточные Web Worker'ы и оптимизации на уровне алгоритмов (Aho-Corasick, WASM) для мгновенной фильтрации и плавной прокрутки файлов с миллионами строк.

## Возможности

### Молниеносная производительность

| Возможность | Описание |
|-------------|----------|
| **Рендеринг на Canvas** | Отображение логов с GPU-ускорением вместо DOM-узлов — обрабатывает миллионы строк без проблем |
| **Виртуальная прокрутка** | Рендерятся только видимые строки. Прокрутка 10M+ строк без задержек |
| **Многопоточная фильтрация** | Параллельные Web Worker'ы распределяют фильтрацию по ядрам CPU |
| **Алгоритм Ахо-Корасик** | Многопаттерновый поиск за O(n+z) — одновременная фильтрация 10+ ключевых слов |
| **Поиск на WebAssembly** | Производительность строкового поиска, близкая к нативной, через WASM-модули |
| **Гибридный умный фильтр** | Автоматический выбор между ripgrep (большие файлы) и JS Worker (маленькие файлы) |
| **Кэширование строк данных** | Повторная фильтрация одного файла пропускает передачу данных — Worker'ам отправляются только ключевые слова |

### Мощная фильтрация и поиск

- **Многоключевая фильтрация** — разделяйте ключевые слова через `|`, экранируйте литеральный пайп через `\|`
- **Поддержка регулярных выражений** — полная поддержка JavaScript RegExp в фильтре и поиске
- **Двухуровневая фильтрация** — первичный фильтр + вторичный фильтр в результатах
- **История фильтров** — постоянная история ключевых слов с нечётким matching (на базе IndexedDB)
- **Подсветка ключевых слов** — 10 предустановленных цветов + произвольный выбор цвета
- **Исключение строк** — правый клик для исключения совпадающих строк из результатов
- **Навигация по поиску** — Enter/Shift+Enter для перехода между совпадениями

### Управление файлами

- **Боковая панель файлового дерева** — перетаскивание файлов, папок или архивов
- **Просмотр архивов** — ZIP, 7z, RAR, tar.gz — просмотр содержимого без распаковки
- **Удалённый файловый сервер** — подключение к удалённым машинам через встроенный C HTTP-сервер (пул потоков, высокая конкурентность)
- **Локальный шаринг** — обмен локальными каталогами с коллегами по LAN
- **Вставка из буфера** — вставка файлов напрямую через Ctrl+V
- **Табличный просмотр CSV/TSV** — парсинг и отображение структурированных данных в сортируемых таблицах
- **Интеграция с Everything** — мгновенный поиск файлов в Windows через Everything HTTP API
- **Интеграция с Ripgrep** — поиск текста в больших файлах в 20-100 раз быстрее

### Визуализация данных

- **Построение CSV-графиков** — интерактивные линейные графики с масштабированием, панорамированием и выбором столбцов
- **Парсер Vlog** — специализированный парсер диагностических логов батарей/устройств (21 поле) с визуализацией
- **Селектор столбцов** — сохранение или удаление определённых столбцов в табличном представлении
- **Экспорт** — копирование отфильтрованных результатов или экспорт в HTML

### Рабочее пространство и продуктивность

- **Мультиоконность** — открывайте несколько файлов логов в отдельных окнах, переключайтесь через Alt+1~9
- **Закладки** — отмечайте важные строки и перемещайтесь между ними
- **Переход к строке** — мгновенный переход к любому номеру строки
- **Быстрые ссылки** — закладки часто используемых веб-сайтов (встроенная веб-панель)
- **ИИ-ассистент** — встроенная панель ИИ-чата для помощи в анализе логов
- **UART последовательный лог** — окно мониторинга логов последовательного порта
- **Масштаб шрифта** — Ctrl+колесо для масштабирования, Alt+колесо для горизонтальной прокрутки
- **Мониторинг системы** — отображение CPU, памяти и памяти приложения в реальном времени
- **Встроенный терминал** — открывайте терминал прямо из приложения

### Технологический стек

| Компонент | Технология |
|-----------|-----------|
| Фреймворк | Electron 28+ |
| Рендеринг | Canvas API (GPU-ускорение) |
| Многопоточность | Web Workers (параллельная фильтрация) |
| Нативный поиск | WebAssembly (компиляция из C) |
| Многопаттерновый matching | Алгоритм Ахо-Корасик |
| Внешний поиск | ripgrep, Everything SDK |
| Удалённый сервер | C + pthread (32 потока, 4096 соединений) |
| Парсинг данных | PapaParse (CSV), пользовательский парсер Vlog |
| Визуализация | Chart.js + плагин масштабирования |
| Хранилище | IndexedDB (история фильтров, закладки) |

## Быстрый старт

### Предварительные требования

- [Node.js](https://nodejs.org/) 18+
- [Electron](https://www.electronjs.org/) 28+
- (Опционально) [7-Zip](https://www.7-zip.org/) для просмотра архивов
- (Опционально) [ripgrep](https://github.com/BurntSushi/ripgrep) для ускоренного поиска
- (Опционально) [Everything](https://www.voidtools.com/) для мгновенного поиска файлов в Windows

### Установка и запуск

```bash
# Клонируйте репозиторий
git clone https://github.com/ranxuefeng2022/ViLog.git
cd ViLog

# Установите зависимости
cd jscode
npm install

# Запустите приложение
npm start
```

### Сборка C-сервера (Опционально — для удалённого просмотра файлов)

```bash
cd server
gcc -o log_server log_server.c -lpthread -O2 -D_GNU_SOURCE

# Запуск на порту 8082
./log_server 8082 /path/to/logs
```

## Тесты производительности

| Сценарий | Строк | Размер файла | Время фильтрации | FPS прокрутки |
|----------|-------|-------------|-------------------|---------------|
| Один файл | 1M | 200 МБ | ~0.3 сек | 60 |
| Многоключевая фильтрация (5 ключевых слов) | 1M | 200 МБ | ~0.5 сек | 60 |
| Объединение 10 файлов | 5M | 1 ГБ | ~1.2 сек | 60 |
| Гибридный режим Ripgrep | 5M | 1 ГБ | ~0.2 сек | 60 |

*Тестовая среда: Intel i7-12700, 32 ГБ RAM, NVMe SSD. Результаты могут отличаться.*

## Варианты использования

- **Встраиваемая/IoT-разработка** — анализ логов устройств, диагностика батарей (формат vlog)
- **Администрирование серверов** — просмотр удалённых логов через встроенный HTTP-сервер
- **QA/тестирование** — сравнение многофайловых логов в параллельных окнах
- **Мобильная разработка** — анализ Android logcat, логов ядра, dmesg
- **Анализ данных** — парсинг CSV/TSV с интерактивной визуализацией графиков
